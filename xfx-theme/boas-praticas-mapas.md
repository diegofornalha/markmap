# Boas Práticas para Mapas Mentais

## Estrutura do Markdown

### Hierarquia Clara
- Use `#` para título principal
- Use `##` para seções principais
- Use `###` para subseções
- Use `-` para tópicos dentro das seções

### Formatação do Conteúdo
- Mantenha textos concisos e diretos
- Use palavras-chave ao invés de frases longas
- Evite redundâncias
- Remova artigos e palavras desnecessárias

### Organização Visual
- Cada conceito em seu próprio nível
- Agrupe informações relacionadas
- Mantenha consistência na estrutura
- Use no máximo 3 níveis de profundidade

## Exemplo de Estrutura

```markdown
# Título Principal

## Seção Principal
### Subseção 1
- Tópico conciso
- Palavra-chave
- Conceito direto

### Subseção 2
- Item específico
- Termo técnico
- Funcionalidade
```

## O Que Evitar

1. **Textos Longos**
   - ❌ "Esta é uma explicação muito longa sobre o conceito"
   - ✅ "Conceito principal"

2. **Redundâncias**
   - ❌ "Sistema de Gerenciamento de Banco de Dados para Gestão"
   - ✅ "Banco de Dados"

3. **Níveis Excessivos**
   - ❌ Mais de 3 níveis de profundidade
   - ✅ Máximo 3 níveis (# > ## > ###)

## Dicas Adicionais

1. **Consistência**
   - Mantenha o mesmo padrão em todo o documento
   - Use a mesma estrutura para seções similares
   - Mantenha a formatação uniforme

2. **Clareza**
   - Priorize termos técnicos precisos
   - Evite abreviações não padrão
   - Use termos conhecidos da área

3. **Visual**
   - Espaçamento adequado entre seções
   - Alinhamento consistente
   - Estrutura limpa e organizada
